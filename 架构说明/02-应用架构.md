# 02 应用架构

## 1. 端到端应用协同

- Web 管理端：面向管理与治理，承载配置、审批、统计分析。
- 大屏监管端：面向校级与院级监管，承载关键指标展示与预警。
- 移动执行端：面向教师与实验员，承载现场操作与反馈留痕。

## 2. 应用能力分层

- 门户层：统一入口、统一身份、统一待办。
- 业务层：计划、资源、过程、安全、统计核心模块。
- 治理层：权限、审计、数据质量、规则引擎。
- 集成层：对接组织、人事、资产、协同办公与消息平台。

## 3. 关键协同机制

- 待办驱动：审批、整改、培训、预警统一流转。
- 状态联动：计划状态与资源状态实时关联。
- 数据联动：过程数据自动进入统计模型与治理模型。

## 4. 应用架构价值

- 提升职业院校实训系统多角色协同效率。
- 提升高校实验室管理系统过程透明度与执行力。
- 提升实训开出率、贵重仪器管理等关键指标达成能力。
